# 异常处理

## 基本语法

```python
try:
    # 可能抛出异常的代码
    result = 10 / 0
except ZeroDivisionError:
    # 捕获特定异常
    result = 0
except (ValueError, TypeError):
    # 同时捕获多种异常
    result = None
except Exception as e:
    # 捕获所有异常（基类）
    print(f"未知错误: {e}")
else:
    # try 中无异常时执行
    print("执行成功")
finally:
    # 无论是否有异常都执行
    print("清理资源")
```

## 常见异常类型

```python
# IndexError：索引越界
lst = [1, 2, 3]
lst[10]      # IndexError: list index out of range

# KeyError：字典键不存在
d = {"a": 1}
d["b"]       # KeyError: 'b'

# TypeError：类型错误
"hello" + 1  # TypeError: can only concatenate str (not "int") to str

# ValueError：值错误
int("abc")   # ValueError: invalid literal for int()

# AttributeError：属性不存在
[].append    # 存在，但 obj.nonexistent 会 AttributeError

# NameError：名称未定义
print(x)     # NameError: name 'x' is not defined

# ImportError：导入失败
import nonexistent_module  # ModuleNotFoundError
```

## 抛出异常

```python
# raise 语句
raise ValueError("无效的值")

# 重新抛出当前异常
try:
    dangerous_operation()
except SomeError:
    # 处理
    raise  # 重新抛出，让上层处理
```

## 自定义异常

```python
class ValidationError(Exception):
    """验证错误异常"""
    def __init__(self, field, message):
        self.field = field
        self.message = message
        super().__init__(f"{field}: {message}")

# 使用
raise ValidationError("email", "格式不正确")

# except 时访问额外属性
try:
    raise ValidationError("age", "必须为正数")
except ValidationError as e:
    print(f"{e.field}: {e.message}")
```

## 异常链

```python
# Python 3 隐式异常链
try:
    int("abc")
except ValueError as e:
    raise RuntimeError("转换失败") from e

# 显式异常链
try:
    int("abc")
except ValueError as e:
    raise RuntimeError("转换失败") from None  # 隐藏原始异常
```

## 异常处理最佳实践

### 1. 具体优先于宽泛

```python
# 不好：太宽泛
try:
    do_something()
except Exception:
    pass

# 好：具体捕获
try:
    do_something()
except ValueError:
    # 只处理 ValueError
    pass
except (TypeError, LookupError):
    # 相关的一起捕获
    pass
```

### 2. 异常不应用于控制流

```python
# 不好：用异常做流程控制
try:
    result = d["key"]
except KeyError:
    result = default_value

# 好：用 get
result = d.get("key", default_value)
```

### 3. 保留异常信息

```python
# 不好：吞掉异常
try:
    do_something()
except SomeError:
    pass  # 完全丢失信息

# 好：记录或重新抛出
try:
    do_something()
except SomeError as e:
    logger.error(f"操作失败: {e}")
    raise
```

### 4. ExceptionGroup（Python 3.11+）

同时抛出多个异常：

```python
# 并发场景下可能同时发生多个错误
def validate_all(data):
    errors = []
    for item in data:
        try:
            validate(item)
        except ValueError as e:
            errors.append(e)

    if errors:
        raise ExceptionGroup("多项验证失败", errors)

# 捕获 ExceptionGroup
try:
    validate_all([...])
except* ValueError as eg:
    # eg 是 ExceptionGroup，包含所有 ValueError
    for e in eg.exceptions:
        print(f"验证错误: {e}")

# except* 和 except 的区别：
# except：只能匹配一种异常
# except*：可以匹配 ExceptionGroup 中的多种异常
```

### 5. 资源清理：try-finally vs with

```python
# try-finally（老式写法）
f = open("file.txt")
try:
    content = f.read()
finally:
    f.close()  # 无论如何都执行

# with 语句（推荐）
with open("file.txt") as f:
    content = f.read()
# 自动关闭，等价于上面的 try-finally

# 为什么 with 更好？
# 1. 更短更清晰
# 2. 即使 __enter__ 中出错也能正确处理
# 3. __exit__ 返回值可以控制异常传播
```

## 使用 traceback 查看详情

```python
import traceback

try:
    1 / 0
except Exception:
    traceback.print_exc()
    # 或获取字符串
    error_info = traceback.format_exc()
```

## 断言 (Assertion)

```python
# assert 条件，条件为 False 时抛出 AssertionError
x = -1
assert x >= 0, "x 必须非负"  # AssertionError: x 必须非负

# 用于开发和测试阶段检查不可能的状态
# 正式环境用 if + raise 更合适
```

## 调试技巧

```python
# 1. print 调试
def buggy_func(x):
    print(f"DEBUG: x = {x}")  # 临时加调试信息
    return x * 2

# 2. breakpoint() (Python 3.7+)
def func():
    x = 1
    breakpoint()  # 进入 pdb debugger
    return x + 1

# 3. pdb 命令
# n (next), s (step into), c (continue)
# p variable (print), l (list), q (quit)
```

## 小结

1. **try/except/else/finally**：完整异常处理结构
2. **异常层次**：从具体异常到 Exception 到 BaseException
3. **具体捕获**：优先捕获具体异常类型
4. **raise**：抛出异常，`raise ... from e` 可以链式传递
5. **自定义异常**：继承 Exception，便于分类处理
6. **异常不做流程控制**：用 if/else 或 dict.get() 代替
