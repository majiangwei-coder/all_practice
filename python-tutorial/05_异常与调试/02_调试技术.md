# 调试技术

## 使用 print 调试

最简单但有效：

```python
def buggy_func(x):
    print(f"DEBUG: x = {x}")  # 临时调试
    result = x * 2
    print(f"DEBUG: result = {result}")
    return result
```

**缺点**：遗留调试代码，发布时需删除。

## assert 断言调试

```python
def divide(a, b):
    assert b != 0, "除数不能为零"  # 条件为 False 时抛出
    return a / b
```

**适用**：检查不应该发生的情况（程序 bug，不是用户输入错误）。

## breakpoint() (Python 3.7+)

```python
def func():
    x = 1
    y = 2
    breakpoint()  # 等价于 import pdb; pdb.set_trace()
    z = x + y
    return z
```

进入 pdb 后常用命令：

| 命令 | 简写 | 说明 |
|------|------|------|
| `help` | `h` | 显示帮助 |
| `next` | `n` | 执行下一行（不进入函数） |
| `step` | `s` | 执行下一行（进入函数） |
| `continue` | `c` | 继续执行到下一个断点 |
| `quit` | `q` | 退出调试器 |
| `print` | `p` | 打印变量值 |
| `list` | `l` | 显示当前代码 |
| `where` | `w` | 显示调用栈 |

## pdb 命令行调试

```bash
python -m pdb script.py
# 或
python -pdb script.py
```

启动后：
- `n` (next) - 下一行
- `s` (step) - 进入函数
- `c` (continue) - 继续
- `p variable` - 打印变量

## pdb.set_trace()

```python
import pdb

def func():
    x = 1
    pdb.set_trace()  # 在此处停下
    y = 2
    return x + y
```

**注意**：Python 3.7+ 推荐用 `breakpoint()`。

## traceback 模块

```python
import traceback

try:
    1 / 0
except Exception:
    # 打印完整回溯
    traceback.print_exc()

    # 获取为字符串
    error_str = traceback.format_exc()
    print(error_str)
```

输出示例：
```
Traceback (most recent call last):
  File "script.py", line 4, in <module>
    1 / 0
ZeroDivisionError: division by zero
```

## logging 日志模块

比 print 更好的调试方式：

```python
import logging

# 配置日志
logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)

logger = logging.getLogger(__name__)

def func():
    logger.debug("进入函数")
    logger.info("信息")
    logger.warning("警告")
    logger.error("错误")
    logger.critical("严重错误")
```

### 日志级别

| 级别 | 值 | 说明 |
|------|-----|------|
| DEBUG | 10 | 详细调试信息 |
| INFO | 20 | 一般信息 |
| WARNING | 30 | 警告（默认级别） |
| ERROR | 40 | 错误 |
| CRITICAL | 50 | 严重错误 |

### 有条件的日志

```python
# 只在 DEBUG 模式计算复杂信息
logger.debug(f"计算结果: {expensive_computation()}")
```

## sys 模块辅助调试

```python
import sys

# 查看对象类型
print(type(x))
print(isinstance(x, list))

# 查看对象所有属性
print(dir(x))

# 查看对象文档
print(x.__doc__)

# 查看调用栈
import traceback
traceback.print_stack()
```

## 常见错误排查

### 1. 变量未定义

```python
# NameError
print(undefined_var)  # NameError

# 检查：变量名拼写、作用域、是否在函数内使用了外层变量
```

### 2. 类型错误

```python
# TypeError
"hello" + 1

# 检查：类型转换、函数参数类型
```

### 3. 索引越界

```python
# IndexError
lst = [1, 2, 3]
print(lst[10])

# 检查：索引范围、动态列表长度
```

### 4. 属性不存在

```python
# AttributeError
obj = "string"
obj.nonexistent

# 检查：属性名拼写、是否导入正确模块
```

### 5. 逻辑错误（最难调试）

```python
# 代码运行不报错，但结果不对
def calc(items):
    total = 0
    for item in items:
        total = item  # 错误：应该是 total += item
    return total

print(calc([1, 2, 3]))  # 3 而不是 6
```

**调试方法**：用 print 或 debugger 逐步检查中间值。

## 单元测试辅助调试

```python
def add(a, b):
    return a + b

# 快速测试
assert add(1, 2) == 3
assert add(0, 0) == 0
assert add(-1, 1) == 0
```

## 小结

1. **print**：简单快速，遗留需删除
2. **assert**：检查不可能的情况
3. **breakpoint()**：Python 3.7+ 内置调试器
4. **pdb**：命令行调试，breakpoint() 底层
5. **IDE 调试器**：可视化更直观
6. **logging**：生产环境调试
7. **traceback**：查看异常详细信息
8. **sys**：查看对象信息（type, dir, __doc__）
