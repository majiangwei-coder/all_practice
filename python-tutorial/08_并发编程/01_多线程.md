# 多线程

> 本章讲解 Python 多线程编程，包括线程基础、同步机制、常见问题。

---

## 1. 线程与进程的区别

### 什么是进程？

**进程**是操作系统分配资源的基本单位。每个进程有独立的内存空间。

```
进程 A                    进程 B
┌─────────┐             ┌─────────┐
│  内存空间  │             │  内存空间  │
│  代码区    │             │  代码区    │
│  数据区    │             │  数据区    │
└─────────┘             └─────────┘
```

### 什么是线程？

**线程**是进程内的执行单元，共享进程的内存空间。

```
进程
┌─────────────────────────┐
│  内存空间（共享）        │
│  ┌─────────┐           │
│  │ 线程 1   │ ← 栈独立  │
│  ├─────────┤           │
│  │ 线程 2   │ ← 栈独立  │
│  └─────────┘           │
│  代码区（共享）          │
│  数据区（共享）          │
└─────────────────────────┘
```

### 对比

| 特性 | 进程 | 线程 |
|------|------|------|
| 内存空间 | 独立 | 共享进程内存 |
| 创建成本 | 高（复制内存） | 低（共享大部分） |
| 通信 | 复杂（IPC） | 简单（共享内存） |
| 并行性 | 真正并行（多核） | 受 GIL 限制 |

## 2. Python 的 GIL 问题（重要！）

### 什么是 GIL？

**GIL (Global Interpreter Lock)**：全局解释器锁。

Python 的解释器（CPython）在任意时刻**只允许一个线程执行 Python 字节码**。

### GIL 的影响

```python
import time

def cpu_task():
    """CPU 密集型任务"""
    total = 0
    for i in range(10**7):
        total += i
    return total

# 测试：单线程
start = time.time()
cpu_task()
cpu_task()
print(f"单线程耗时: {time.time() - start:.2f}s")
```

对于 CPU 密集型任务，多线程**不能加速**，因为 GIL 限制。

### 为什么需要 GIL？

历史原因 + 简化实现。GIL 使得单进程内的线程安全更容易实现。

### 什么时候多线程有用？

**I/O 密集型任务**！当线程等待 I/O（如网络请求、文件读写）时，Python 会释放 GIL，其他线程可以执行。

```python
# 多线程对 I/O 密集型任务有用
import threading
import time

def fetch_url(url):
    """模拟网络请求"""
    time.sleep(1)  # 等待 I/O
    return url

# 单线程串行：3秒
start = time.time()
for url in ["url1", "url2", "url3"]:
    fetch_url(url)
print(f"串行耗时: {time.time() - start:.2f}s")  # ~3s

# 多线程并行：1秒
start = time.time()
threads = []
for url in ["url1", "url2", "url3"]:
    t = threading.Thread(target=fetch_url, args=(url,))
    threads.append(t)
    t.start()

for t in threads:
    t.join()

print(f"并行耗时: {time.time() - start:.2f}s")  # ~1s
```

## 3. threading 模块基础

### 创建线程

```python
import threading
import time

def task(n):
    """线程要执行的任务"""
    print(f"任务 {n} 开始")
    time.sleep(1)
    print(f"任务 {n} 完成")


# 方法1：创建 Thread 对象
t = threading.Thread(target=task, args=(1,))
t.start()   # 启动线程（异步执行）
t.join()    # 等待线程结束

print("主线程继续执行")


# 方法2：继承 Thread 类
class MyThread(threading.Thread):
    def __init__(self, n):
        super().__init__()
        self.n = n

    def run(self):  # 线程执行的内容
        print(f"任务 {self.n} 开始")
        time.sleep(1)
        print(f"任务 {self.n} 完成")


t = MyThread(2)
t.start()
t.join()
```

### 线程池（推荐方式）

手动管理线程繁琐，用线程池更方便：

```python
from concurrent.futures import ThreadPoolExecutor

def task(n):
    return n * 2


# 使用 with 语句，自动管理资源
with ThreadPoolExecutor(max_workers=4) as executor:
    # 方式1：submit + result
    future1 = executor.submit(task, 1)
    future2 = executor.submit(task, 2)
    print(f"结果: {future1.result()}, {future2.result()}")

    # 方式2：map（更简洁）
    results = executor.map(task, range(5))
    print(list(results))  # [0, 2, 4, 6, 8]
```

## 4. 线程同步：Lock

多线程共享资源时，需要**锁**来防止竞争条件。

### 竞争条件示例

```python
# 不使用锁：结果不确定
counter = 0

def increment():
    global counter
    for _ in range(100000):
        counter += 1  # 这一步不是原子操作！

# 两个线程同时修改 counter
t1 = threading.Thread(target=increment)
t2 = threading.Thread(target=increment)
t1.start()
t2.start()
t1.join()
t2.join()

# 由于竞争条件，结果通常小于 200000
print(f"counter = {counter}")  # 可能小于 200000！
```

### 图解竞争条件

```
线程 A 读取 counter (0)
线程 B 读取 counter (0)
线程 A 计算 counter + 1 (1)
线程 B 计算 counter + 1 (1)
线程 A 写入 counter (1)
线程 B 写入 counter (1)  ← 覆盖了 A 的结果！
```

### 使用 Lock

```python
counter = 0
lock = threading.Lock()  # 创建锁

def increment():
    global counter
    for _ in range(100000):
        with lock:  # 获取锁，结束后自动释放
            counter += 1

# 两个线程
t1 = threading.Thread(target=increment)
t2 = threading.Thread(target=increment)
t1.start()
t2.start()
t1.join()
t2.join()

print(f"counter = {counter}")  # 200000 ✓
```

### with lock 的原理

```python
lock.acquire()  # 获取锁
try:
    counter += 1  # 临界区代码
finally:
    lock.release()  # 释放锁

# 等价于：
with lock:
    counter += 1
```

## 5. 其他同步机制

### RLock：可重入锁

同一个线程可以多次获取的锁：

```python
lock = threading.RLock()

lock.acquire()  # 第1次获取
lock.acquire()  # 第2次获取（不会死锁！）
print("获取了2次")
lock.release()  # 释放1次
lock.release()  # 释放2次

# 普通 Lock 会死锁：
# lock = threading.Lock()
# lock.acquire()
# lock.acquire()  # 死锁！自己等自己
```

### Semaphore：信号量

控制同时访问的线程数量：

```python
import threading
import time

# 允许3个线程同时访问资源
semaphore = threading.Semaphore(3)

def access_resource(n):
    semaphore.acquire()  # 获取信号
    print(f"线程 {n} 开始访问")
    time.sleep(1)
    print(f"线程 {n} 结束访问")
    semaphore.release()  # 释放信号


# 10个线程抢3个信号
threads = [threading.Thread(target=access_resource, args=(i,)) for i in range(10)]
for t in threads:
    t.start()
for t in threads:
    t.join()
```

### Event：事件

线程间信号通知：

```python
import threading
import time

event = threading.Event()

def waiter():
    print("等待信号...")
    event.wait()  # 阻塞，等待信号
    print("收到信号，继续执行！")


def setter():
    print("发送者准备...")
    time.sleep(2)
    print("发送信号！")
    event.set()  # 发送信号


t1 = threading.Thread(target=waiter)
t2 = threading.Thread(target=setter)
t1.start()
t2.start()
t1.join()
t2.join()
```

### Condition：条件变量

更复杂的线程通信：

```python
import threading

condition = threading.Condition()
items = []
MAX_ITEMS = 5

def producer():
    for i in range(10):
        with condition:
            while len(items) >= MAX_ITEMS:
                condition.wait()  # 等待消费者消费
            items.append(i)
            print(f"生产了 {i}")
            condition.notify_all()  # 通知消费者


def consumer():
    while True:
        with condition:
            while not items:
                condition.wait()  # 等待生产者生产
            item = items.pop(0)
            print(f"消费了 {item}")
            condition.notify_all()
        if item == 9:  # 消费完就退出
            break


t1 = threading.Thread(target=producer)
t2 = threading.Thread(target=consumer)
t1.start()
t2.start()
t1.join()
t2.join()
```

## 6. 线程间通信：Queue

线程安全的队列：

```python
import queue
import threading
import time

q = queue.Queue()  # 线程安全队列


def producer():
    for i in range(5):
        q.put(i)
        print(f"生产: {i}")
        time.sleep(0.5)


def consumer():
    while True:
        item = q.get()  # 阻塞等待
        if item is None:  # 结束信号
            break
        print(f"消费: {item}")
        q.task_done()  # 标记完成


t1 = threading.Thread(target=producer)
t2 = threading.Thread(target=consumer)
t1.start()
t2.start()
t1.join()
q.put(None)  # 发送结束信号
t2.join()
```

## 7. 守护线程

主线程结束时，守护线程自动终止：

```python
import threading
import time

def background_task():
    """后台任务"""
    while True:
        print("后台运行中...")
        time.sleep(1)


# daemon=True 表示这是守护线程
t = threading.Thread(target=background_task, daemon=True)
t.start()

time.sleep(3)
print("主线程结束，程序退出")
# 守护线程会自动被终止
```

## 8. 常见问题

### 问题1：死锁

```python
# 错误：两把锁，不同顺序获取
lock1 = threading.Lock()
lock2 = threading.Lock()

def task1():
    lock1.acquire()
    lock2.acquire()  # 可能被 task2 阻塞！
    # ...
    lock2.release()
    lock1.release()

def task2():
    lock2.acquire()
    lock1.acquire()  # 可能被 task1 阻塞！
    # ...
    lock1.release()
    lock2.release()

# 解决：所有线程按相同顺序获取锁
```

### 问题2：线程数设置

```python
import os

# I/O 密集型：可以多一些（等待 I/O 时释放 GIL）
# 一般设为 2 * CPU 核心数
thread_count = os.cpu_count() * 2

# CPU 密集型：不宜过多（受 GIL 限制）
# 一般等于 CPU 核心数
thread_count = os.cpu_count()
```

## 9. 小结

### 必须掌握

1. **GIL 限制**：Python 多线程不能真正并行，适合 I/O 密集型
2. **threading.Thread**：创建线程
3. **ThreadPoolExecutor**：线程池，更方便
4. **Lock**：互斥锁，保护共享资源
5. **Queue**：线程间通信

### 何时用多线程

| 场景 | 选择 |
|------|------|
| CPU 密集型 | 多进程 |
| I/O 密集型（网络请求） | 多线程或 asyncio |
| CPU + I/O 混合 | 多进程 + 多线程 |
