# 多进程

> 本章讲解 Python 多进程编程，包括进程基础、进程间通信、共享内存、进程池。

---

## 1. 为什么需要多进程

### GIL 的限制

Python 的 GIL 使得**多线程无法真正并行**执行 CPU 密集型任务：

```python
# 多线程对 CPU 密集型任务无效
import threading
import time

def cpu_task():
    total = 0
    for i in range(10**7):
        total += i
    return total

# 双线程并不比单线程快
start = time.time()
t1 = threading.Thread(target=cpu_task)
t2 = threading.Thread(target=cpu_task)
t1.start()
t2.start()
t1.join()
t2.join()
print(f"双线程耗时: {time.time() - start:.2f}s")
# 由于 GIL，两个线程无法同时执行
```

### 多进程解决 GIL 问题

**多进程**可以绕过 GIL，每个进程有独立的 Python 解释器和内存空间：

```python
import multiprocessing as mp

def cpu_task():
    total = 0
    for i in range(10**7):
        total += i
    return total

if __name__ == "__main__":
    start = time.time()

    # 多进程：每个进程有独立 GIL，真正并行
    with mp.Pool(2) as pool:
        results = pool.map(cpu_task, [1, 2])

    print(f"多进程耗时: {time.time() - start:.2f}s")
    # 接近单进程的一半时间（2核）
```

## 2. 进程 vs 线程

| 特性 | 多进程 | 多线程 |
|------|--------|--------|
| 内存空间 | 独立 | 共享 |
| GIL 限制 | 无（每个进程独立 GIL） | 有 |
| 并行能力 | 真正并行（多核） | 受 GIL 限制 |
| 创建成本 | 高 | 低 |
| 通信方式 | Queue / Pipe / 共享内存 | 共享变量 |
| 稳定性 | 高（进程隔离） | 低（一个崩溃影响全部） |
| 适用场景 | CPU 密集型 | I/O 密集型 |

## 3. multiprocessing 模块基础

### Process 类

```python
import multiprocessing as mp
import os

def worker(name):
    """子进程执行的任务"""
    print(f"进程 {name} 开始，PID: {os.getpid()}")
    # os.getpid() 获取当前进程 ID
    return name * 2


if __name__ == "__main__":
    # 创建进程
    p = mp.Process(target=worker, args=("A",))
    p.start()   # 启动进程
    p.join()    # 等待进程结束

    print("主进程继续执行")
```

### 继承 Process 类

```python
import multiprocessing as mp

class Worker(mp.Process):
    def __init__(self, name):
        super().__init__()
        self.name = name
        self.result = None

    def run(self):
        """进程执行的内容"""
        self.result = self.name * 2
        print(f"Worker {self.name} 完成")


if __name__ == "__main__":
    w = Worker("B")
    w.start()
    w.join()
    print(w.result)  # 'BB'
```

### 进程池 Pool

管理多个进程更简单的方式：

```python
import multiprocessing as mp

def square(n):
    return n * n

if __name__ == "__main__":
    # 创建进程池（4个进程）
    with mp.Pool(4) as pool:
        # map：将任务分配给进程池执行
        results = pool.map(square, range(10))
        print(results)  # [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]
```

**注意**：Windows 系统必须使用 `if __name__ == "__main__"` 保护，否则会递归创建进程！

## 4. 进程间通信

进程内存独立，不能直接共享变量，需要 IPC（进程间通信）机制。

### Queue：队列

```python
import multiprocessing as mp

def producer(queue):
    """生产者：向队列放数据"""
    for i in range(5):
        queue.put(i)
        print(f"生产: {i}")
    queue.put(None)  # 发送结束信号


def consumer(queue):
    """消费者：从队列取数据"""
    while True:
        item = queue.get()
        if item is None:
            break
        print(f"消费: {item}")


if __name__ == "__main__":
    queue = mp.Queue()  # 进程安全的队列

    p1 = mp.Process(target=producer, args=(queue,))
    p2 = mp.Process(target=consumer, args=(queue,))

    p1.start()
    p2.start()

    p1.join()
    p2.join()
```

### Pipe：管道

```python
import multiprocessing as mp

def sender(conn):
    """发送者"""
    conn.send("Hello")
    conn.send("World")
    conn.close()  # 关闭连接


def receiver(conn):
    """接收者"""
    while True:
        try:
            msg = conn.recv()  # 接收消息
            print(f"收到: {msg}")
        except EOFError:
            break  # 发送端已关闭


if __name__ == "__main__":
    # Pipe() 返回两个连接对象，分别用于两端
    parent_conn, child_conn = mp.Pipe()

    p1 = mp.Process(target=sender, args=(child_conn,))
    p2 = mp.Process(target=receiver, args=(parent_conn,))

    p1.start()
    p2.start()
    p1.join()
    p2.join()
```

### Queue vs Pipe

| 特性 | Queue | Pipe |
|------|-------|------|
| 使用难度 | 简单 | 简单 |
| 线程安全 | 是 | 否（需加锁） |
| 速度 | 较慢 | 较快 |
| 多端通信 | 支持多生产/多消费 | 仅两端 |
| 推荐场景 | 通用场景 | 简单双向通信 |

## 5. 共享内存

### Value 和 Array

基本类型的共享内存：

```python
import multiprocessing as mp

def increment(counter, data):
    """多个进程共享 counter 和 data"""
    for _ in range(100):
        with counter.get_lock():  # 需要锁来保护
            counter.value += 1

    # 修改共享数组
    for i in range(len(data)):
        data[i] *= 2


if __name__ == "__main__":
    # 'i' 表示有符号整数，'d' 表示双精度浮点
    shared_counter = mp.Value('i', 0)  # 共享整数
    shared_array = mp.Array('i', [1, 2, 3])  # 共享数组

    # 10 个进程同时累加
    processes = [
        mp.Process(target=increment, args=(shared_counter, shared_array))
        for _ in range(10)
    ]

    for p in processes:
        p.start()
    for p in processes:
        p.join()

    print(f"计数器: {shared_counter.value}")  # 1000
    print(f"数组: {list(shared_array)}")  # [2, 4, 6] (翻倍了10次)
```

**类型代码**：

| 代码 | 类型 | 大小 |
|------|------|------|
| `'b'` | 有符号字节 | 1 |
| `'B'` | 无符号字节 | 1 |
| `'i'` | 有符号整数 | 4 |
| `'I'` | 无符号整数 | 4 |
| `'f'` | 浮点数 | 4 |
| `'d'` | 双精度浮点 | 8 |

### Manager：高级共享

Manager 可以创建更复杂的共享对象：

```python
import multiprocessing as mp

def worker(d, l, n):
    """操作共享字典和列表"""
    d[n] = n * n
    l.append(n)


if __name__ == "__main__":
    with mp.Manager() as manager:
        # 创建共享对象
        shared_dict = manager.dict()
        shared_list = manager.list()

        # 启动多个进程操作共享对象
        processes = [
            mp.Process(target=worker, args=(shared_dict, shared_list, i))
            for i in range(5)
        ]

        for p in processes:
            p.start()
        for p in processes:
            p.join()

        print(dict(shared_dict))  # {0: 0, 1: 1, 2: 4, 3: 9, 4: 16}
        print(list(shared_list))  # [0, 1, 2, 3, 4]
```

Manager 支持的共享类型：
- `dict()`：共享字典
- `list()`：共享列表
- `Namespace()`：共享命名空间
- `Queue()`：共享队列
- `Lock()` / `RLock()` / `Semaphore()` / `Event()` / `Condition()`

> **注意**：Manager 比 Value/Array 慢，但使用更灵活。性能敏感场景用 Value/Array。

## 6. 进程池高级用法

### map / starmap

```python
import multiprocessing as mp

def square(n):
    return n * n

def power(base, exp):
    return base ** exp

if __name__ == "__main__":
    with mp.Pool(4) as pool:
        # map：单个参数
        results = pool.map(square, range(5))
        print(results)  # [0, 1, 4, 9, 16]

        # starmap：多个参数（解包）
        params = [(2, 1), (2, 2), (2, 3)]
        results = pool.starmap(power, params)
        print(results)  # [2, 4, 8]
```

### apply / apply_async

```python
import multiprocessing as mp

def square(n):
    return n * n

def callback(result):
    """回调函数：异步完成时调用"""
    print(f"结果: {result}")

if __name__ == "__main__":
    with mp.Pool(4) as pool:
        # apply：阻塞等待结果
        result = pool.apply(square, (5,))
        print(f"同步结果: {result}")

        # apply_async：非阻塞，立即返回
        async_result = pool.apply_async(square, (6,))
        # 做其他事...
        print("等待结果中...")
        result = async_result.get()  # 获取结果（可带超时）
        print(f"异步结果: {result}")

        # 带回调的异步调用
        pool.apply_async(square, (7,), callback=callback)
        pool.close()  # 不再接受新任务
        pool.join()   # 等待所有完成
```

### imap / imap_unordered

```python
if __name__ == "__main__":
    with mp.Pool(4) as pool:
        # imap：惰性求值，边处理边返回
        for result in pool.imap(square, range(10)):
            print(result)

        # imap_unordered：不保证顺序，更快
        for result in pool.imap_unordered(square, range(10)):
            print(result)
```

## 7. concurrent.futures 接口

更高级的统一接口，与线程池 API 一致：

```python
from concurrent.futures import ProcessPoolExecutor
import time

def cpu_task(n):
    """CPU 密集型任务"""
    total = 0
    for i in range(10**7):
        total += i
    return n, total


# 使用 ProcessPoolExecutor（API 与 ThreadPoolExecutor 完全相同）
with ProcessPoolExecutor(max_workers=4) as executor:
    # submit：提交单个任务
    future = executor.submit(cpu_task, 1)
    print(future.result())

    # map：批量提交
    results = executor.map(cpu_task, range(4))
    for n, total in results:
        print(f"任务 {n} 完成")
```

## 8. 常见问题

### 问题1：if __name__ == "__main__" 保护

```python
# Windows 系统必须！否则递归创建进程导致错误
import multiprocessing as mp

def worker():
    print("工作")

# 必须加保护！
if __name__ == "__main__":
    p = mp.Process(target=worker)
    p.start()
    p.join()
```

### 问题2：进程数设置

```python
import os

# CPU 密集型：等于 CPU 核心数
cpu_count = os.cpu_count()
pool = mp.Pool(cpu_count)

# I/O 密集型：可以更多（如 2倍）
pool = mp.Pool(cpu_count * 2)

# 默认值：os.cpu_count()
pool = mp.Pool()
```

### 问题3：序列化限制

```python
# 进程间传递的参数必须可 pickle 序列化
def worker(data):
    return data * 2

# 错误：lambda 不能序列化
# with mp.Pool(4) as pool:
#     pool.map(lambda x: x * 2, range(5))  # PicklingError！

# 正确：用普通函数
def double(x):
    return x * 2

with mp.Pool(4) as pool:
    pool.map(double, range(5))
```

### 问题4：避免共享状态

```python
# 错误：全局变量不跨进程
counter = 0  # 每个进程有独立副本！

def increment():
    global counter
    counter += 1

# 正确：使用共享内存
def increment(shared_counter):
    with shared_counter.get_lock():
        shared_counter.value += 1
```

## 9. 实际应用

### 并行文件处理

```python
import multiprocessing as mp
import os

def count_lines(filename):
    """统计文件行数"""
    with open(filename, 'r', encoding='utf-8') as f:
        return filename, sum(1 for _ in f)


def process_files(file_list):
    """并行统计多个文件的行数"""
    with mp.Pool(os.cpu_count()) as pool:
        results = pool.map(count_lines, file_list)

    for filename, lines in results:
        print(f"{filename}: {lines}行")

    return sum(lines for _, lines in results)


if __name__ == "__main__":
    files = ["file1.txt", "file2.txt", "file3.txt"]
    total = process_files(files)
    print(f"总行数: {total}")
```

### 并行计算

```python
import multiprocessing as mp

def is_prime(n):
    """判断素数"""
    if n < 2:
        return False
    for i in range(2, int(n ** 0.5) + 1):
        if n % i == 0:
            return False
    return True

def find_primes_in_range(start, end):
    """查找区间的素数"""
    return [n for n in range(start, end) if is_prime(n)]


if __name__ == "__main__":
    # 将大范围拆分成多个小范围并行计算
    total_range = 100000
    num_workers = 4
    chunk_size = total_range // num_workers

    ranges = [
        (i * chunk_size, (i + 1) * chunk_size)
        for i in range(num_workers)
    ]

    with mp.Pool(num_workers) as pool:
        results = pool.starmap(find_primes_in_range, ranges)

    # 合并结果
    all_primes = []
    for primes in results:
        all_primes.extend(primes)

    print(f"找到 {len(all_primes)} 个素数")
```

## 10. 小结

### 必须掌握

1. **多进程解决 GIL 问题**：每个进程独立 GIL，适合 CPU 密集型
2. **Process / Pool**：创建进程的基本方式
3. **Queue / Pipe**：进程间通信
4. **Value / Array / Manager**：共享内存
5. **`if __name__ == "__main__"`**：Windows 必须的保护
6. **concurrent.futures.ProcessPoolExecutor**：统一接口

### 选择指南

| 场景 | 方案 | 原因 |
|------|------|------|
| CPU 密集型 | 多进程 | 真正并行，绕过 GIL |
| I/O 密集型 | 多线程 / asyncio | 创建开销小 |
| 大量数据共享 | 多线程 | 共享内存更方便 |
| 稳定性要求高 | 多进程 | 进程隔离，互不影响 |
| 简单并行任务 | ProcessPoolExecutor | API 简洁 |

### 常见错误

1. **忘记 `if __name__` 保护**：Windows 上会递归创建进程
2. **使用不可序列化的参数**：进程间传递需要 pickle
3. **全局变量不共享**：每个进程有独立副本
4. **进程数过多**：超过 CPU 核心数反而降低性能
5. **锁使用不当**：共享内存需要加锁保护
