# 测试基础

> "代码没测试就是有 bug 的代码。" 本章教你写测试——不是可选的额外工作，而是开发的一部分。

---

## 1. 为什么写测试

### 没有测试的开发流程

```
写代码 → 手动点点 → 看起来OK → 提交
            ↓
       后来改了某处
            ↓
       💥 之前的功能坏了！（回归bug）
            ↓
       没人发现，直到用户投诉
```

### 有测试的开发流程

```
写代码 → 写测试 → 跑测试 → 全绿 → 提交
            ↓
       后来改了某处 → 跑测试
            ↓
       某个测试红了！→ 立即发现 → 修复 → 提交
```

## 2. 最简单的测试：assert

```python
# test_math.py

def add(a, b):
    return a + b

def multiply(a, b):
    return a * b

# 测试（用 assert）
assert add(2, 3) == 5, "加法测试失败"
assert add(-1, 1) == 0, "负数加法测试失败"
assert multiply(3, 4) == 12, "乘法测试失败"
assert multiply(0, 100) == 0, "零乘法测试失败"

print("所有测试通过！")
```

**assert 的局限**：第一个失败就停止，错误信息不详细。

## 3. unittest 标准库

### 基本使用

```python
import unittest
from my_module import add, multiply


class TestMathFunctions(unittest.TestCase):
    """测试数学函数"""

    def test_add_positive(self):
        """测试正数加法"""
        self.assertEqual(add(2, 3), 5)

    def test_add_negative(self):
        """测试负数加法"""
        self.assertEqual(add(-1, 1), 0)

    def test_add_zero(self):
        """测试加零"""
        self.assertEqual(add(5, 0), 5)

    def test_multiply(self):
        """测试乘法"""
        self.assertEqual(multiply(3, 4), 12)
        self.assertEqual(multiply(0, 5), 0)
        self.assertEqual(multiply(-2, 3), -6)


if __name__ == "__main__":
    unittest.main()
```

运行：

```bash
python test_math.py
# ....
# Ran 4 tests in 0.001s
# OK
```

### unittest 常用断言

```python
class TestExample(unittest.TestCase):

    def test_assertions(self):
        # 相等
        self.assertEqual(a, b)          # a == b
        self.assertNotEqual(a, b)       # a != b

        # 真假
        self.assertTrue(condition)       # condition is True
        self.assertFalse(condition)      # condition is False

        # None
        self.assertIsNone(x)
        self.assertIsNotNone(x)

        # 身份
        self.assertIs(a, b)             # a is b
        self.assertIsNot(a, b)

        # 包含
        self.assertIn(item, container)   # item in container
        self.assertNotIn(item, container)

        # 异常
        with self.assertRaises(ValueError):
            int("abc")  # 预期抛出 ValueError

        with self.assertRaises(ZeroDivisionError):
            x = 1 / 0

        # 近似
        self.assertAlmostEqual(0.1 + 0.2, 0.3)  # 浮点数比较
```

### setUp 和 tearDown

```python
class TestDatabase(unittest.TestCase):

    def setUp(self):
        """每个测试方法前运行"""
        self.db = Database(":memory:")
        self.db.create_tables()
        self.db.insert({"id": 1, "name": "Alice"})

    def tearDown(self):
        """每个测试方法后运行"""
        self.db.close()

    def test_query(self):
        result = self.db.query(1)
        self.assertEqual(result["name"], "Alice")

    def test_insert(self):
        self.db.insert({"id": 2, "name": "Bob"})
        self.assertEqual(self.db.count(), 2)
```

## 4. pytest（推荐！）

pytest 是更现代、更简洁的测试框架。

### 安装

```bash
pip install pytest
```

### 第一个 pytest 测试

```python
# test_calculator.py
import pytest
from calculator import add, multiply, divide


def test_add():
    assert add(2, 3) == 5
    assert add(-1, 1) == 0
    assert add(0, 0) == 0


def test_multiply():
    assert multiply(3, 4) == 12
    assert multiply(-2, 3) == -6


def test_divide():
    assert divide(10, 2) == 5
    with pytest.raises(ValueError, match="除数不能为零"):
        divide(10, 0)
```

运行：

```bash
pytest                          # 测试当前目录
pytest test_calculator.py       # 测试特定文件
pytest -v                       # 详细输出
pytest -v --tb=short            # 简短回溯
pytest -x                       # 第一个失败就停止
pytest -k "test_add"            # 只运行名称包含 test_add 的
```

### pytest 特点

```python
# 1. 不需要写类，自由写函数
def test_something():
    assert func() == expected

# 2. 参数化测试
@pytest.mark.parametrize("a, b, expected", [
    (2, 3, 5),
    (-1, 1, 0),
    (0, 0, 0),
    (100, 200, 300),
])
def test_add_parametrized(a, b, expected):
    assert add(a, b) == expected

# 3. 更友好的断言失败信息
# pytest 会自动显示：什么值不相等、为什么
```

### fixture：测试前准备数据

```python
import pytest

@pytest.fixture
def sample_user():
    """创建测试用的用户数据"""
    return {"id": 1, "name": "Alice", "age": 30}

@pytest.fixture
def database():
    """创建测试数据库，测试后自动清理"""
    db = Database(":memory:")
    db.create_tables()
    yield db          # ← 测试用这个
    db.close()        # ← 测试后清理


def test_user_name(sample_user):
    assert sample_user["name"] == "Alice"

def test_database_insert(database):
    database.insert({"id": 1, "name": "Bob"})
    assert database.count() == 1
```

## 5. Mock：模拟外部依赖

```python
from unittest.mock import Mock, patch
import pytest

# 模拟函数调用
def test_with_mock():
    mock_api = Mock()
    mock_api.get_user.return_value = {"name": "Alice"}

    result = mock_api.get_user(1)
    assert result == {"name": "Alice"}
    mock_api.get_user.assert_called_once_with(1)


# 替换真实对象
@patch("my_module.requests.get")
def test_fetch_data(mock_get):
    # 配置模拟对象
    mock_get.return_value.status_code = 200
    mock_get.return_value.json.return_value = {"data": "test"}

    from my_module import fetch_data
    result = fetch_data("http://api.example.com/data")

    assert result == {"data": "test"}
    mock_get.assert_called_once()

# 模拟时间
@patch("time.time")
def test_with_time(mock_time):
    mock_time.return_value = 1234567890.0
    assert time.time() == 1234567890.0
```

## 7. 什么值得测试

```
需要测试的：
✅ 核心业务逻辑          → 计算、验证、转换
✅ API 接口              → 请求处理、返回格式
✅ 边界条件              → 空列表、0、None、极大值
✅ 错误处理              → 异常情况是否正确抛出

可以不测试的：
❌ 简单的 getter/setter   → @property 只是返回 self._x
❌ 第三方库的功能          → 它们有自己的测试
❌ 配置文件                → 除非有复杂解析逻辑
```

## 8. 测试的原则

### AAA 模式

```python
def test_transfer():
    # Arrange（准备）—— 设置测试数据
    sender = Account(balance=1000)
    receiver = Account(balance=0)

    # Act（执行）—— 执行被测试的操作
    result = transfer(sender, receiver, 500)

    # Assert（断言）—— 验证结果
    assert result is True
    assert sender.balance == 500
    assert receiver.balance == 500
```

### 给测试起好名字

```python
# 好的测试名
def test_transfer_insufficient_balance_returns_false(): ...
def test_divide_by_zero_raises_value_error(): ...

# 不好的测试名
def test_transfer1(): ...
def test_error(): ...
```

## 9. TDD 测试驱动开发

```
Red → Green → Refactor

1. Red:    写一个失败的测试
2. Green:  写最少代码让测试通过
3. Refactor: 重构代码，保持测试通过
```

```python
# 步骤1：写测试（Red）
def test_calculator_add():
    calc = Calculator()
    assert calc.add(2, 3) == 5

# 步骤2：让测试通过（Green）
class Calculator:
    def add(self, a, b):
        return a + b

# 步骤3：加新功能，循环
def test_calculator_subtract():
    calc = Calculator()
    assert calc.subtract(5, 3) == 2
```

## 11. 小结

1. **assert**：最简单的测试，适合快速验证
2. **unittest**：Python 自带，功能齐全
3. **pytest**：推荐！更简洁、更强大
4. **fixture**：准备和清理测试数据
5. **参数化**：一组输入 → 一组测试
6. **Mock**：隔离外部依赖
7. **覆盖率**：知道哪些代码没测到
8. **写测试是习惯**：不是负担，是保障
