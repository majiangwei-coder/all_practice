# 虚拟环境与包管理

> 每个 Python 项目都应该有独立的虚拟环境！避免不同项目间的依赖冲突。本章是新手到专业开发者的必经之路。

---

## 1. 为什么需要虚拟环境

### 没有虚拟环境的灾难

```
项目A 需要 requests==2.25.0
项目B 需要 requests==2.31.0

如果都装在全局环境：
  pip install requests==2.25.0  → 安装了 2.25.0
  pip install requests==2.31.0  → 覆盖为 2.31.0
  项目A 不兼容 → 💥 炸了！
```

### 虚拟环境解决一切

```
项目A/
├── .venv/              ← 独立环境，requests 2.25.0
└── ...

项目B/
├── .venv/              ← 独立环境，requests 2.31.0
└── ...

两个项目的依赖完全隔离，互不影响。
```

---

## 2. 创建虚拟环境

### 方式1：venv（Python 自带，推荐）

```bash
# Windows
python -m venv .venv

# Linux/Mac
python3 -m venv .venv
```

### 方式2：指定 Python 版本

```bash
# 如果有多个 Python 版本
py -3.12 -m venv .venv        # Windows
python3.12 -m venv .venv      # Linux/Mac
```

### 激活环境

```bash
# Windows (CMD)
.venv\Scripts\activate

# Windows (PowerShell)
.venv\Scripts\Activate.ps1

# Linux/Mac
source .venv/bin/activate
```

激活后，终端前面会出现 `(.venv)` 标识：

```
(.venv) $ python --version
(.venv) $ pip list
```

### 退出环境

```bash
deactivate
```

## 3. pip 包管理

### 安装包

```bash
# 安装最新版本
pip install requests

# 安装特定版本
pip install requests==2.31.0

# 安装版本范围
pip install "requests>=2.25,<3.0"

# 从 requirements.txt 安装
pip install -r requirements.txt
```

### 查看包

```bash
# 列出已安装的包
pip list

# 查看某个包的详细信息
pip show requests

# 检查哪些包需要更新
pip list --outdated

# 查看包依赖树
pip install pipdeptree
pipdeptree
```

### 卸载包

```bash
pip uninstall requests
```

### 升级 pip

```bash
# Windows
python -m pip install --upgrade pip

# Linux/Mac
pip install --upgrade pip
```

## 4. 依赖管理

### requirements.txt（传统方式）

```txt
# requirements.txt
requests==2.31.0
flask>=3.0,<4.0
numpy>=1.24
pandas
```

生成和安装：

```bash
# 导出当前环境的依赖
pip freeze > requirements.txt

# 从文件安装
pip install -r requirements.txt
```

### pip freeze 的问题

`pip freeze` 会导出**所有**包（包括间接依赖），文件很长：

```
# 你可能只装了 requests，但 freeze 会导出：
certifi==2024.2.2
charset-normalizer==3.3.2
idna==3.6
requests==2.31.0
urllib3==2.2.1
# ... 这些都是自动安装的依赖
```

### pyproject.toml（现代方式）

```toml
# pyproject.toml
[project]
name = "myapp"
version = "0.1.0"
requires-python = ">=3.10"
dependencies = [
    "requests>=2.31",
    "click>=8.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=7.0",
    "black>=24.0",
    "mypy>=1.0",
]
test = [
    "pytest>=7.0",
    "pytest-cov>=4.0",
]

# 安装
# pip install -e .       安装项目（开发模式）
# pip install -e ".[dev]" 安装项目 + 开发依赖
```

## 5. pip 常用技巧

### 国内镜像加速

```bash
# 临时使用
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple requests

# 永久配置
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

常用国内镜像：
- 清华：`https://pypi.tuna.tsinghua.edu.cn/simple`
- 阿里云：`https://mirrors.aliyun.com/pypi/simple/`
- 中科大：`https://pypi.mirrors.ustc.edu.cn/simple/`

### 安装本地包

```bash
# 从本地目录安装（开发模式，修改代码不需重装）
pip install -e .

# 从 wheel 文件安装
pip install ./dist/myapp-0.1.0-py3-none-any.whl

# 从 GitHub 安装
pip install git+https://github.com/user/repo.git
```

### 检查依赖冲突

```bash
pip check  # 检查已安装包之间有无冲突
```

## 6. 现代工具：uv / Poetry

### uv（极速 Pip 替代品）

```bash
# 安装 uv
pip install uv

# 创建虚拟环境
uv venv

# 安装包（比 pip 快 10-100 倍）
uv pip install requests flask numpy pandas

# 同步依赖
uv pip sync requirements.txt
```

### Poetry（全功能项目管理）

```bash
# 安装 Poetry
pip install poetry

# 新建项目
poetry new my-project

# 添加依赖
poetry add requests
poetry add --dev pytest black

# 安装所有依赖
poetry install

# 运行命令
poetry run python main.py
poetry run pytest
```

**Poetry vs pip + venv：**
- Poetry 同时管理依赖 + 虚拟环境 + 打包
- pip + venv 更简单更轻量
- 团队成员少/项目简单 → pip + venv
- 专业项目/多人协作 → Poetry 或 uv

## 7. 虚拟环境的 .gitignore

永远不要把虚拟环境提交到 Git！

```gitignore
# .gitignore
.venv/
venv/
env/
__pycache__/
*.pyc
*.pyo
*.egg-info/
dist/
build/
```

在项目的 `.gitignore` 中添加 `.venv/`，每次在新机器上：

```bash
git clone <repo>
cd <repo>
python -m venv .venv
source .venv/bin/activate  # 或 .venv\Scripts\activate
pip install -r requirements.txt  # 或 pip install -e .
```

## 8. pipenv（另一种选择）

```bash
pip install pipenv

# 创建环境 + 安装包
pipenv install requests
pipenv install --dev pytest

# 激活环境
pipenv shell

# 运行命令
pipenv run python main.py
```

pipenv 自动管理 Pipfile 和 Pipfile.lock。

## 9. 项目初始化检查清单

```bash
# 1. 确认 Python 版本
python --version  # 应 >= 3.9

# 2. 创建虚拟环境
python -m venv .venv

# 3. 激活环境
source .venv/bin/activate  # Linux/Mac
# .venv\Scripts\activate   # Windows

# 4. 升级 pip
pip install --upgrade pip

# 5. 创建 requirements.txt 或 pyproject.toml
# 添加项目依赖

# 6. 安装依赖
pip install -r requirements.txt

# 7. 创建 .gitignore
echo ".venv/" >> .gitignore

# 8. 开始写代码！
```

## 10. 故障排除

### pip install 权限错误

```bash
# 错误: Permission denied
# 原因: 试图安装到系统目录

# 解决: 确保在虚拟环境中
which python   # 应指向 .venv/
which pip      # 应指向 .venv/
```

### 虚拟环境激活不了（Windows PowerShell）

```powershell
# 错误: 无法加载文件 Activate.ps1

# 解决: 更改执行策略
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 包找不到

```bash
# pip install xxx 报错: No matching distribution found

# 检查:
# 1. 包名拼写正确？
# 2. Python 版本兼容？（某些包不支持太旧/太新的 Python）
# 3. 平台兼容？（某些包只有 Linux 版本）
```

## 11. 小结

1. **每个项目一个虚拟环境**：隔离依赖，避免冲突
2. **venv**：Python 自带，够用
3. **激活/退出**：`source .venv/bin/activate` / `deactivate`
4. **requirements.txt**：记录精确依赖版本
5. **pyproject.toml**：现代化的项目配置
6. **uv / Poetry**：更高级的依赖管理工具
7. **虚拟环境不要提交 Git**：加到 .gitignore
8. **国内用镜像**：加速下载
