# 项目结构与配置

> 从单文件脚本到大型项目，良好的项目结构让代码可维护、可扩展。本章教你组织 Python 项目的标准方式。

---

## 1. 从脚本到项目

### 阶段1：单文件脚本

```
hello.py           ← 所有代码在一个文件
```

适合：一次性脚本、简单工具

### 阶段2：多文件模块

```
my_project/
├── main.py        ← 入口
├── utils.py       ← 工具函数
├── models.py      ← 数据模型
└── config.py      ← 配置
```

适合：小程序

### 阶段3：标准项目结构

```
my_project/
├── my_package/              ← 源码包
│   ├── __init__.py
│   ├── main.py              ← 入口
│   ├── core/
│   │   ├── __init__.py
│   │   ├── engine.py
│   │   └── config.py
│   ├── utils/
│   │   ├── __init__.py
│   │   ├── helpers.py
│   │   └── validators.py
│   ├── models/
│   │   ├── __init__.py
│   │   └── user.py
│   └── api/
│       ├── __init__.py
│       └── routes.py
├── tests/                   ← 测试
│   ├── __init__.py
│   ├── conftest.py
│   ├── test_core.py
│   └── test_utils.py
├── docs/                    ← 文档
├── scripts/                 ← 辅助脚本
├── .venv/                   ← 虚拟环境（不提交）
├── .gitignore
├── pyproject.toml           ← 项目配置
├── README.md
└── LICENSE
```

## 2. src-layout vs flat-layout

### flat-layout（src 和项目根同级）

```
my_project/
├── my_project/     ← 代码在这里
│   ├── __init__.py
│   └── ...
├── tests/
└── pyproject.toml
```

### src-layout（源码在 src/ 下）

```
my_project/
├── src/
│   └── my_project/ ← 代码在这里
│       ├── __init__.py
│       └── ...
├── tests/
└── pyproject.toml
```

**src-layout 优势**：强制用 `pip install -e .` 安装后才能运行，避免导入路径问题。

## 4. 配置管理

### 方式1：环境变量（推荐）

```python
# config.py
import os

class Config:
    """应用配置，从环境变量读取"""

    # 数据库
    DB_HOST = os.getenv("DB_HOST", "localhost")
    DB_PORT = int(os.getenv("DB_PORT", "5432"))
    DB_NAME = os.getenv("DB_NAME", "myapp")
    DB_USER = os.getenv("DB_USER", "postgres")
    DB_PASSWORD = os.getenv("DB_PASSWORD", "")  # 无默认值

    # 应用
    DEBUG = os.getenv("DEBUG", "false").lower() == "true"
    SECRET_KEY = os.getenv("SECRET_KEY", "dev-secret")

    # API
    API_TIMEOUT = int(os.getenv("API_TIMEOUT", "30"))


# 使用
config = Config()
print(config.DB_HOST)  # localhost（或用环境变量覆盖）
```

### .env 文件（开发时使用）

```bash
# .env
DB_HOST=localhost
DB_PORT=5432
DB_PASSWORD=dev-password
DEBUG=true
SECRET_KEY=dev-secret-key
```

```python
# 使用 python-dotenv 加载 .env
# pip install python-dotenv

from dotenv import load_dotenv
import os

load_dotenv()  # 加载 .env 文件到环境变量

# 之后用 os.getenv() 读取即可
db_host = os.getenv("DB_HOST", "localhost")
```

**重要**：`.env` 文件不要提交到 Git！

```gitignore
.env
.env.local
.env.*.local
```

### 方式2：YAML/JSON 配置文件

```yaml
# config.yaml
database:
  host: localhost
  port: 5432
  name: myapp

app:
  debug: true
  log_level: INFO

api:
  base_url: https://api.example.com
  timeout: 30
```

```python
# config.py
import yaml  # pip install pyyaml
from pathlib import Path

def load_config(config_path: str = "config.yaml") -> dict:
    """加载 YAML 配置文件"""
    path = Path(config_path)
    if path.exists():
        with open(path, encoding="utf-8") as f:
            return yaml.safe_load(f)
    return {}

config = load_config()
print(config["database"]["host"])  # localhost
```

### 方式3：dataclass 配置

```python
from dataclasses import dataclass, field
from pathlib import Path
import json

@dataclass
class AppConfig:
    """类型安全的配置"""
    debug: bool = False
    secret_key: str = "change-me"
    db_url: str = "sqlite:///app.db"
    workers: int = 4
    allowed_hosts: list[str] = field(default_factory=lambda: ["localhost"])

    @classmethod
    def from_env(cls):
        """从环境变量加载"""
        import os
        return cls(
            debug=os.getenv("DEBUG", "false").lower() == "true",
            secret_key=os.getenv("SECRET_KEY", "change-me"),
            db_url=os.getenv("DATABASE_URL", "sqlite:///app.db"),
            workers=int(os.getenv("WORKERS", "4")),
        )

    @classmethod
    def from_json(cls, path: str):
        """从 JSON 文件加载"""
        with open(path) as f:
            return cls(**json.load(f))
```

## 5. 多环境配置

```python
# config.py
import os

class BaseConfig:
    """基础配置"""
    SECRET_KEY = os.getenv("SECRET_KEY", "default-secret")

class DevelopmentConfig(BaseConfig):
    """开发环境"""
    DEBUG = True
    DB_URL = "sqlite:///dev.db"
    LOG_LEVEL = "DEBUG"

class TestingConfig(BaseConfig):
    """测试环境"""
    DEBUG = False
    DB_URL = "sqlite:///test.db"
    LOG_LEVEL = "WARNING"

class ProductionConfig(BaseConfig):
    """生产环境"""
    DEBUG = False
    DB_URL = os.getenv("DATABASE_URL")  # 必须从环境变量获取
    LOG_LEVEL = "WARNING"

# 根据环境变量选择配置
config_map = {
    "development": DevelopmentConfig,
    "testing": TestingConfig,
    "production": ProductionConfig,
}

env = os.getenv("APP_ENV", "development")
Config = config_map.get(env, DevelopmentConfig)
```

## 6. __init__.py 模式

```python
# my_package/__init__.py
"""
my_package - 项目主包

暴露主要的公共 API。
"""

__version__ = "0.1.0"

# 选择性导入，让外部使用更方便
from my_package.core.engine import Engine
from my_package.models.user import User

# from my_package import * 时导出的内容
__all__ = ["Engine", "User", "__version__"]
```

```python
# 子包 __init__.py
# my_package/core/__init__.py
from .engine import Engine
from .config import Config

__all__ = ["Engine", "Config"]
```

## 7. 入口点（Entry Points）

### 单文件入口

```python
# main.py
def main():
    print("Hello, World!")

if __name__ == "__main__":
    main()
```

```bash
python main.py
```

### 可安装的命令行工具

```toml
# pyproject.toml
[project.scripts]
myapp = "my_package.cli:main"
```

```python
# my_package/cli.py
import click  # pip install click

@click.group()
def main():
    """MyApp - 命令行工具"""
    pass

@main.command()
@click.argument("name")
def greet(name):
    """打招呼"""
    click.echo(f"Hello, {name}!")

@main.command()
@click.option("--port", default=8000, help="端口号")
def serve(port):
    """启动服务"""
    click.echo(f"服务已启动，端口: {port}")

if __name__ == "__main__":
    main()
```

安装后：

```bash
pip install -e .
myapp greet Alice     # Hello, Alice!
myapp serve --port 8080
```

## 10. 小结

1. **从小开始**：单文件 → 多文件 → 标准结构，按需演进
2. **pyproject.toml**：现代化项目配置，一个文件管理所有
3. **配置与代码分离**：环境变量 > YAML > 硬编码
4. **`.env` 不提交 Git**：敏感信息用环境变量
5. **多环境配置**：开发/测试/生产分开
6. **入口点**：让项目可安装为命令行工具
7. **合理分层**：core/utils/models/api 各司其职
