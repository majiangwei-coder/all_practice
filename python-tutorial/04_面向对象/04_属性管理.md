# 属性管理：@property、描述符与 __slots__

> 控制属性的访问、修改和删除是面向对象编程的重要部分。本章深入讲解 Python 的属性管理机制。

---

## 1. 为什么需要属性管理

### 问题场景

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age     # 直接暴露，无法验证！

p = Person("Alice", -5)   # 年龄是负数？不合理
p.age = "三十岁"           # 年龄变成字符串？类型错误
```

我们希望：
- 设置 age 时自动验证（必须 > 0）
- 读取 age 时自动格式化
- 禁止直接修改某些属性

---

## 2. 传统方式：getter/setter

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self._age = None   # 内部存储
        self.set_age(age)  # 通过 setter 设置

    def get_age(self):
        return self._age

    def set_age(self, value):
        if not isinstance(value, int):
            raise TypeError("年龄必须是整数")
        if value < 0:
            raise ValueError("年龄不能为负数")
        self._age = value

p = Person("Alice", 30)
print(p.get_age())    # 30
p.set_age(25)
# p.set_age(-5)      # ValueError!

# 缺点：语法不自然
# p.age = 25  ← 想要这样写，但不能！
```

**太啰嗦了！** 每次访问都要调用方法。

---

## 3. @property 装饰器

@property 让你用**属性访问的语法**，实际上是**调用方法**：

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self._age = age    # 注意：内部用 _age 存储

    @property
    def age(self):
        """获取年龄（像访问属性一样）"""
        print("获取年龄...")  # 可以加逻辑
        return self._age

    @age.setter
    def age(self, value):
        """设置年龄（像赋值一样）"""
        print(f"设置年龄为 {value}")
        if not isinstance(value, int):
            raise TypeError("年龄必须是整数")
        if value < 0:
            raise ValueError("年龄不能为负数")
        self._age = value

    @age.deleter
    def age(self):
        """删除年龄"""
        print("删除年龄")
        del self._age


# 使用：语法完全自然！
p = Person("Alice", 30)
print(p.age)        # 获取年龄... → 30（实际调用的是方法！）
p.age = 25          # 设置年龄为 25（实际调用的是 setter！）
# p.age = -5        # ValueError: 年龄不能为负数
del p.age           # 删除年龄
```

### 图解 @property 机制

```
p.age = 25          ← 你写的是属性赋值

  ↓ Python 实际执行 ↓

p.age(25)           ← @age.setter 装饰的方法

p.age               ← 你写的是属性访问

  ↓ Python 实际执行 ↓

p.age()             ← @property 装饰的方法
```

## 4. @property 的实用场景

### 场景1：计算属性（只读）

```python
class Rectangle:
    def __init__(self, width, height):
        self.width = width
        self.height = height

    @property
    def area(self):
        """面积 = 宽 × 高（动态计算）"""
        return self.width * self.height

    @property
    def perimeter(self):
        """周长 = 2 × (宽 + 高)"""
        return 2 * (self.width + self.height)

    @property
    def is_square(self):
        """判断是否为正方形"""
        return self.width == self.height

r = Rectangle(5, 3)
print(r.area)       # 15（像属性，实际是计算出来的）
print(r.perimeter)  # 16
print(r.is_square)  # False

r.width = 10
print(r.area)       # 30（自动反映最新值！）
```

### 场景2：数据验证

```python
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self._balance = balance

    @property
    def balance(self):
        return self._balance

    @balance.setter
    def balance(self, amount):
        if amount < 0:
            raise ValueError("余额不能为负数")
        self._balance = amount

    @property
    def balance_formatted(self):
        """格式化显示余额"""
        return f"¥{self._balance:,.2f}"

    def deposit(self, amount):
        """存款"""
        self.balance += amount  # 触发 setter 验证

    def withdraw(self, amount):
        """取款"""
        if amount > self.balance:
            raise ValueError("余额不足")
        self.balance -= amount

account = BankAccount("Alice", 1000)
print(account.balance_formatted)  # ¥1,000.00
account.deposit(500)
print(account.balance_formatted)  # ¥1,500.00
```

### 场景3：延迟加载

```python
class DataLoader:
    def __init__(self, filename):
        self.filename = filename
        self._data = None  # 还没加载

    @property
    def data(self):
        """首次访问时才加载数据"""
        if self._data is None:
            print(f"正在加载 {self.filename}...")
            with open(self.filename, 'r') as f:
                self._data = f.read()
        return self._data

loader = DataLoader("huge_file.txt")
# 此时还没有读取文件！
# print(loader.data)   ← 第一次访问时才加载
# print(loader.data)   ← 直接返回缓存的数据
```

## 5. 属性装饰器进阶

### 只读属性（不定义 setter）

```python
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self):
        return self._radius
    # 没有 setter → 只读！
    # c.radius = 10  # AttributeError!

    def set_radius(self, value):
        """通过方法修改（给了明确信号：这有副作用）"""
        if value <= 0:
            raise ValueError("半径必须为正数")
        self._radius = value
```

### property 函数形式（旧式）

```python
# 与 @property 装饰器等价，但不推荐
class Person:
    def __init__(self, age):
        self._age = age

    def _get_age(self):
        return self._age

    def _set_age(self, value):
        self._age = value

    age = property(_get_age, _set_age)  # 旧式写法
```

## 6. 描述符（Descriptor）

描述符是 @property 背后的底层机制。当你需要**在多个类中复用**属性逻辑时使用。

### 描述符协议

```python
# 描述符是实现了以下任意方法的类：
# __get__(self, instance, owner)  — 获取属性时调用
# __set__(self, instance, value)  — 设置属性时调用
# __delete__(self, instance)      — 删除属性时调用
```

### 自定义描述符：类型验证

```python
class TypedField:
    """描述符：确保属性是特定类型"""

    def __init__(self, field_type, default=None):
        self.field_type = field_type
        self.default = default

    def __set_name__(self, owner, name):
        """Python 3.6+：自动获取属性名"""
        self.name = name

    def __get__(self, instance, owner):
        if instance is None:
            return self  # 类级别访问返回描述符本身
        return instance.__dict__.get(self.name, self.default)

    def __set__(self, instance, value):
        if not isinstance(value, self.field_type):
            raise TypeError(f"{self.name} 必须是 {self.field_type.__name__} 类型")
        instance.__dict__[self.name] = value


class Person:
    name = TypedField(str)        # name 必须是字符串
    age = TypedField(int, 0)      # age 必须是整数，默认0

    def __init__(self, name, age):
        self.name = name          # 触发 __set__
        self.age = age            # 触发 __set__

p = Person("Alice", 30)
# p.age = "三十"  # TypeError: age 必须是 int 类型
# p.name = 123    # TypeError: name 必须是 str 类型
```

### 自定义描述符：范围验证

```python
class RangeField:
    """描述符：确保数值在指定范围内"""

    def __init__(self, min_value=None, max_value=None):
        self.min_value = min_value
        self.max_value = max_value

    def __set_name__(self, owner, name):
        self.name = name

    def __get__(self, instance, owner):
        if instance is None:
            return self
        return instance.__dict__.get(self.name)

    def __set__(self, instance, value):
        if self.min_value is not None and value < self.min_value:
            raise ValueError(f"{self.name} 不能小于 {self.min_value}")
        if self.max_value is not None and value > self.max_value:
            raise ValueError(f"{self.name} 不能大于 {self.max_value}")
        instance.__dict__[self.name] = value


class Product:
    price = RangeField(min_value=0)          # 价格 >= 0
    discount = RangeField(min_value=0, max_value=100)  # 折扣 0~100

    def __init__(self, price, discount=0):
        self.price = price
        self.discount = discount

p = Product(99.9, 20)
# p.price = -5      # ValueError!
# p.discount = 150  # ValueError!
```

### 缓存描述符

```python
class CachedProperty:
    """描述符：属性值只计算一次，之后返回缓存"""

    def __init__(self, func):
        self.func = func
        self.name = func.__name__

    def __get__(self, instance, owner):
        if instance is None:
            return self
        # 计算并缓存
        value = self.func(instance)
        instance.__dict__[self.name] = value  # 缓存到实例字典
        return value


class DataAnalyzer:
    def __init__(self, data):
        self.data = data

    @CachedProperty
    def analysis_result(self):
        """昂贵计算，只执行一次"""
        print("正在分析数据...（这个计算很费时）")
        import time
        time.sleep(2)  # 模拟耗时计算
        return sum(self.data) / len(self.data)

a = DataAnalyzer([1, 2, 3, 4, 5])
print(a.analysis_result)  # 正在分析数据... → 3.0（计算）
print(a.analysis_result)  # 3.0（直接返回缓存，不再计算！）
```

## 7. __slots__ 节省内存

默认每个实例有一个 `__dict__` 字典存储属性，占用内存。`__slots__` 用更紧凑的方式存储属性。

```python
import sys

class NormalPerson:
    def __init__(self, name, age):
        self.name = name
        self.age = age

class SlottedPerson:
    __slots__ = ['name', 'age']  # 预定义属性列表

    def __init__(self, name, age):
        self.name = name
        self.age = age


p1 = NormalPerson("Alice", 30)
p2 = SlottedPerson("Bob", 25)

# 内存对比
print(sys.getsizeof(p1.__dict__))  # 约 104 字节
# p2 没有 __dict__，节省了内存！

# 不能动态添加属性
# p2.city = "NYC"  # AttributeError!
```

### __slots__ 的优缺点

| 优点 | 缺点 |
|------|------|
| 节省内存（无 `__dict__`） | 不能动态添加属性 |
| 属性访问稍快 | 不能多继承自不同 `__slots__` 类 |
| 防止属性名拼写错误 | 不能使用 `@cached_property`（需 `__dict__`） |

### 何时用 __slots__

```python
# 场景：创建大量实例
# 比如一个游戏中成千上万的粒子和子弹
class Bullet:
    __slots__ = ['x', 'y', 'vx', 'vy', 'damage']

    def __init__(self, x, y, vx, vy, damage):
        self.x = x
        self.y = y
        self.vx = vx
        self.vy = vy
        self.damage = damage

# 100万个子弹：
# 有 __slots__：约 56MB
# 无 __slots__：约 168MB（多了 __dict__ 的开销）
```

## 8. 属性访问的完整流程

当你写 `obj.attr` 时，Python 按以下顺序查找：

```
1. 数据描述符（有 __set__ 的描述符） — 在类的 __dict__ 中查找
2. 实例的 __dict__
3. 非数据描述符（只有 __get__ 的描述符）
4. 类的 __dict__
5. 父类的 __dict__（按 MRO 顺序）
6. __getattr__() — 兜底方法
```

```python
# 验证查找顺序
class DataDescriptor:
    def __get__(self, instance, owner):
        return "数据描述符"
    def __set__(self, instance, value):
        pass  # 有 __set__ → 数据描述符

class NonDataDescriptor:
    def __get__(self, instance, owner):
        return "非数据描述符"
    # 没有 __set__ → 非数据描述符

class MyClass:
    data_desc = DataDescriptor()
    non_data_desc = NonDataDescriptor()

    def __init__(self):
        self.data_desc = "实例属性"     # 被数据描述符覆盖！
        self.non_data_desc = "实例属性"  # 覆盖非数据描述符

obj = MyClass()
print(obj.data_desc)      # "数据描述符"（优先级最高）
print(obj.non_data_desc)  # "实例属性"（实例覆盖了非数据描述符）
```

## 9. 对比总结

| 方式 | 适用场景 | 复用性 | 复杂度 |
|------|---------|--------|--------|
| 直接属性 | 无需验证的简单属性 | - | 低 |
| `_protected` | 约定不直接访问 | - | 低 |
| `@property` | 单个类的属性管理 | 低（不跨类） | 中 |
| 描述符 | 跨多个类复用的属性逻辑 | **高** | 高 |
| `__slots__` | 大量实例的内存优化 | - | 低 |

---

## 10. 小结

1. **@property**：让方法像属性一样调用，语法优雅
2. **@xxx.setter**：设置属性时自动验证/转换
3. **@xxx.deleter**：删除属性时清理资源
4. **只读属性**：不定义 setter 即可
5. **描述符**：跨类复用属性逻辑的底层机制
6. **`__slots__`**：节省内存，禁止动态属性
7. **访问顺序**：数据描述符 > 实例 `__dict__` > 非数据描述符 > 类 `__dict__`
