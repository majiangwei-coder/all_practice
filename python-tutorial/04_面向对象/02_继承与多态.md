# 继承与多态

## 继承基础

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        raise NotImplementedError("子类必须实现 speak 方法")

class Dog(Animal):
    def speak(self):
        return f"{self.name} says woof!"

class Cat(Animal):
    def speak(self):
        return f"{self.name} says meow!"
```

### 图示

```
Animal (父类/基类)
    ├── __init__(self, name)
    └── speak(self)

    ├── Dog (子类)
    │       └── speak(self)  → "Buddy says woof!"
    │
    └── Cat (子类)
            └── speak(self)  → "Whiskers says meow!"
```

## 多继承

```python
class Flyer:
    def fly(self):
        return "flying"

class Swimmer:
    def swim(self):
        return "swimming"

class Duck(Animal, Flyer, Swimmer):
    pass

duck = Duck("Donald")
print(duck.speak())  # "Donald says quack!"
print(duck.fly())    # "flying"
print(duck.swim())   # "swimming"
```

### MRO (Method Resolution Order)

当多个父类有同名方法时，按 MRO 顺序查找：

```python
class A:
    def test(self):
        return "A"

class B(A):
    pass

class C(A):
    def test(self):
        return "C"

class D(B, C):
    pass

print(D().test())   # "C" (按 MRO 顺序)
print(D.__mro__)    # 查看完整 MRO
```

## super()：调用父类方法

```python
class Animal:
    def __init__(self, name):
        self.name = name

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)  # 调用父类 __init__
        self.breed = breed      # 新增属性

    def speak(self):
        return f"{self.name} says woof!"

dog = Dog("Buddy", "Golden Retriever")
print(dog.name)    # "Buddy"
print(dog.breed)   # "Golden Retriever"
```

### super 的原理

```python
# super() 相当于 super(Class, self)
class Child(Parent):
    def __init__(self):
        super(Child, self).__init__()  # 等价于 super().__init__()
```

## isinstance() 与 issubclass()

```python
class Animal:
    pass

class Dog(Animal):
    pass

dog = Dog()

isinstance(dog, Dog)      # True
isinstance(dog, Animal)    # True (Dog 是 Animal 的子类)
isinstance(dog, object)   # True (所有类的祖先)

issubclass(Dog, Animal)    # True
issubclass(Animal, Dog)    # False
```

## 多态的本质

**同一接口，不同实现**：

```python
def makeSpeak(animal):
    """只需知道 animal 有 speak 方法，不关心具体类型"""
    return animal.speak()

animals = [Dog("Buddy"), Cat("Whiskers"), Dog("Rex")]

for animal in animals:
    print(makeSpeak(animal))
# Buddy says woof!
# Whiskers says meow!
# Rex says woof!
```

### 鸭子类型 (Duck Typing)

Python 的多态是**鸭子类型**：不关心类型，只关心行为：

```python
class Duck:
    def fly(self):
        return "Duck flying"

class Airplane:
    def fly(self):
        return "Airplane flying"

def let_fly(obj):
    return obj.fly()

# 只要有 fly 方法就能用，不需继承关系
let_fly(Duck())      # "Duck flying"
let_fly(Airplane())  # "Airplane flying"
```

## 抽象基类 (ABC)

定义必须被子类实现的接口：

```python
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        """子类必须实现"""
        pass

    @abstractmethod
    def perimeter(self):
        pass

class Rectangle(Shape):
    def __init__(self, w, h):
        self.w = w
        self.h = h

    def area(self):
        return self.w * self.h

    def perimeter(self):
        return 2 * (self.w + self.h)

# 不能实例化抽象类
# shape = Shape()  # TypeError!

rect = Rectangle(3, 4)
print(rect.area())      # 12
print(rect.perimeter()) # 14
```

## 组合 vs 继承

```python
# 继承：is-a 关系
class Dog(Animal):
    # Dog is an Animal
    pass

# 组合：has-a 关系
class Owner:
    def __init__(self, name):
        self.name = name
        self.pets = []

    def add_pet(self, pet):
        self.pets.append(pet)

# 优先组合：更灵活，避免继承层次过深
```

## Mixin 模式

Mixin 是一种"混入"额外功能的轻量类：

```python
class LogMixin:
    """混入日志功能"""
    def log(self, message):
        print(f"[{self.__class__.__name__}] {message}")

class SerializeMixin:
    """混入序列化功能"""
    def to_dict(self):
        return {k: v for k, v in self.__dict__.items() if not k.startswith('_')}

    def to_json(self):
        import json
        return json.dumps(self.to_dict(), ensure_ascii=False)


# 普通业务类，通过多重继承混入功能
class User(LogMixin, SerializeMixin):
    def __init__(self, name, email):
        self.name = name
        self.email = email

    def save(self):
        self.log(f"保存用户: {self.name}")
        # 保存逻辑...

class Product(LogMixin, SerializeMixin):
    def __init__(self, title, price):
        self.title = title
        self.price = price

# 使用
user = User("Alice", "alice@example.com")
user.save()                    # [User] 保存用户: Alice
print(user.to_dict())          # {'name': 'Alice', 'email': 'alice@example.com'}
print(user.to_json())          # {"name": "Alice", "email": "alice@example.com"}

product = Product("Laptop", 9999)
print(product.to_dict())       # {'title': 'Laptop', 'price': 9999}
```

### Mixin 原则

1. Mixin 类名通常以 `Mixin` 结尾
2. Mixin 不应该有 `__init__`（或调用 `super().__init__` 保持 MRO 链）
3. Mixin 只提供额外功能，不定义核心行为
4. 功能独立、可组合

## 属性拦截：`__getattr__` vs `__getattribute__`

```python
class LazyLoader:
    """属性不存在时才加载"""

    def __init__(self):
        self._loaded = {}

    def __getattr__(self, name):
        """只在属性不在 __dict__ 中时触发"""
        if name.startswith('_'):
            raise AttributeError(name)
        print(f"延迟加载: {name}")
        value = f"加载了 {name}"  # 模拟加载
        self._loaded[name] = value
        return value

obj = LazyLoader()
print(obj.data)    # 延迟加载: data → 加载了 data
print(obj.data)    # 加载了 data（这次从 __dict__ 取，不触发 __getattr__）

# __getattribute__：每次属性访问都触发（慎用！易造成递归）
class Logged:
    def __getattribute__(self, name):
        print(f"访问属性: {name}")
        return super().__getattribute__(name)  # 必须调用父类

obj = Logged()
obj.x = 1
print(obj.x)  # 访问属性: x
```

## 小结

1. **继承**：子类自动获得父类的属性和方法
2. **super()**：调用父类方法，用于初始化和扩展
3. **多继承**：按 MRO 顺序查找方法
4. **多态**：同一接口不同实现，Python 用鸭子类型实现
5. **抽象基类 (ABC)**：定义必须实现的方法接口
6. **组合 vs 继承**：优先组合，更灵活
