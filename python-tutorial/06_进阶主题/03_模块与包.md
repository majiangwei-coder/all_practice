# 模块与包

> 模块是 Python 组织代码的基本方式。把一个大文件拆成多个小文件，用 import 连接起来。本章从 import 开始，到构建自己的包结束。

---

## 1. 模块是什么

**模块就是一个 .py 文件。** 每个 .py 文件都是一个模块。

```
my_project/
├── main.py          ← 一个模块
├── utils.py         ← 另一个模块
└── config.py        ← 又一个模块
```

---

## 2. import 导入

### 基本导入方式

```python
# 方式1：导入整个模块
import math
print(math.sqrt(16))   # 4.0
print(math.pi)         # 3.14159...

# 方式2：导入特定内容
from math import sqrt, pi
print(sqrt(16))        # 4.0（直接用，不用 math.）
print(pi)              # 3.14159...

# 方式3：导入并起别名
import numpy as np
import pandas as pd

# 方式4：导入全部（不推荐）
from math import *     # 污染命名空间，不知道哪些名字来自 math
```

### import 的工作原理

```python
# 当你写 import math 时，Python 做了这些事：
# 1. 在 sys.modules 中查找 'math' 是否存在
# 2. 如果不存在，查找并执行 math.py（或内置模块）
# 3. 将模块对象存储在 sys.modules 中（下次 import 直接用）
# 4. 在当前命名空间中创建 'math' 这个名字指向模块对象
```

**重要**：每个模块只被导入一次！重复 import 不会重新执行。

```python
# module_a.py
print("模块 A 被加载了！")
x = 1

# main.py
import module_a   # 输出：模块 A 被加载了！
import module_a   # 无输出（已经导入过了）
import module_a   # 无输出
```

### 重新加载模块（开发时用）

```python
import importlib
importlib.reload(module_a)  # 强制重新执行模块代码
```

## 3. 模块搜索路径

当 `import something` 时，Python 去哪里找？

```python
import sys
print(sys.path)  # 搜索路径列表
# ['当前目录', 'PYTHONPATH环境变量', '标准库目录', 'site-packages']
```

搜索顺序（从前到后）：
1. **当前目录**（脚本所在目录）
2. **PYTHONPATH** 环境变量
3. **标准库目录**（如 `/usr/lib/python3.12/`）
4. **site-packages**（pip 安装的第三方库）

```python
# 如果遇到 ModuleNotFoundError：
# 1. 检查文件名是否正确
# 2. 检查文件是否在搜索路径中
# 3. 打印 sys.path 查看

import sys
sys.path.append("/path/to/your/module")  # 临时添加
```

## 4. `__name__` 与 `if __name__ == "__main__"`

每个模块有一个 `__name__` 属性：

```python
# utils.py
def helper():
    return "工具函数"

print(f"utils.py 的 __name__ = {__name__}")

if __name__ == "__main__":
    # 这段代码只在直接运行 utils.py 时执行
    # import utils 时不执行
    print("utils.py 被直接运行了！")
    print(helper())
```

```python
# main.py
import utils  # 输出：utils.py 的 __name__ = utils （不是 __main__）
              # if __name__ == "__main__" 的代码不执行

print(utils.helper())  # 工具函数
```

**规则**：
- 直接运行 `python utils.py` → `__name__` = `"__main__"`
- import 导入 → `__name__` = 模块文件名 `"utils"`

### 标准模式

```python
# 每个可执行脚本的标准写法
def main():
    # 真正的程序逻辑
    print("程序开始")
    # ...

if __name__ == "__main__":
    main()
```

这样脚本既可以作为模块被导入，也可以直接运行。

## 5. 相对导入与绝对导入

### 绝对导入（推荐）

从项目根目录开始的完整路径：

```python
# 假设项目结构：
# my_project/
# ├── main.py
# ├── utils/
# │   ├── __init__.py
# │   ├── file_utils.py
# │   └── string_utils.py
# └── models/
#     ├── __init__.py
#     └── user.py

# main.py
from utils.file_utils import read_file     # 绝对导入
from models.user import User                # 绝对导入
import utils.string_utils as str_utils      # 绝对导入
```

### 相对导入

在**包内部**使用点号引用：

```python
# utils/file_utils.py 内部引用同包的其他模块
from . import string_utils        # . 表示当前包（utils/）
from .string_utils import clean  # 从同包导入
from ..models import User         # .. 表示上级目录

# 注意：相对导入只能在包内使用
# 直接运行 python utils/file_utils.py 会报错
# 因为相对导入需要知道"自己在哪个包里"
```

### 对比

| 特点 | 绝对导入 | 相对导入 |
|------|---------|----------|
| 可读性 | 高（路径完整） | 中（需知上下文） |
| 重构 | 需改路径 | 相对路径不变 |
| 适用场景 | 外部使用 | 包内部模块间引用 |
| 推荐度 | 优先使用 | 仅在包内部使用 |

## 6. 包（Package）

**包是包含 `__init__.py` 文件的目录。**

### `__init__.py` 的作用

```python
# my_package/
# ├── __init__.py      ← 必须！标识这是 Python 包
# ├── module_a.py
# └── module_b.py

# __init__.py 可以：
# 1. 空文件（标识此目录为包）
# 2. 定义包级别的变量和函数
# 3. 控制 from package import * 的行为
```

### 使用 `__init__.py` 组织 API

```python
# my_package/__init__.py
"""
my_package - 示例包

提供数据处理和用户管理功能。
"""

__version__ = "1.0.0"
__author__ = "Your Name"

# 从子模块导入，让用户方便使用
from .module_a import DataProcessor
from .module_b import UserManager

# 控制 from my_package import * 导出的内容
__all__ = ["DataProcessor", "UserManager", "__version__"]

# 包级别的初始化代码
print(f"加载 my_package v{__version__}")
```

```python
# 用户使用
import my_package
print(my_package.__version__)  # 1.0.0

# 可以直接用，不用知道内部结构
from my_package import DataProcessor
processor = DataProcessor()
```

### `__all__` 控制导出

```python
# my_package/__init__.py
__all__ = ["public_func", "PublicClass"]

def public_func():
    pass

def _internal_func():  # 下划线开头的不被导出
    pass

class PublicClass:
    pass

# 用户使用
from my_package import *  # 只导入 __all__ 中的内容
```

## 7. 命名空间包（Python 3.3+）

没有 `__init__.py` 的目录也是包（隐式命名空间包）：

```
namespace_pkg/       ← 不需要 __init__.py！
├── sub_pkg_a/
│   └── module.py
└── sub_pkg_b/
    └── module.py
```

用于**跨多个独立分发包**组织代码：

```python
# 公司有两独立项目
# project-core/
#   company/
#     core/
#       __init__.py
#       engine.py

# project-plugins/
#   company/              ← 两个目录同名
#     plugins/
#       __init__.py
#       extra.py

# 使用时，它们自动合并为一个 company 包
from company.core import engine
from company.plugins import extra
```

## 8. 循环导入问题

### 问题示例

```python
# module_a.py
from module_b import func_b

def func_a():
    return "A" + func_b()

# module_b.py
from module_a import func_a  # ← 此时 module_a 还没执行完！

def func_b():
    return "B" + func_a()
```

运行结果：`ImportError` 或 `AttributeError`

### 解决方案

```python
# 方案1：延迟导入（在函数内部 import）
# module_a.py
def func_a():
    from module_b import func_b   # 使用时再导入
    return "A" + func_b()

# module_b.py
def func_b():
    from module_a import func_a
    return "B" + func_a()

# 方案2：导入模块而非函数
# module_a.py
import module_b

def func_a():
    return "A" + module_b.func_b()

# 方案3：重构，提取共同依赖
# common.py
def shared_logic():
    pass

# module_a.py → import common
# module_b.py → import common
```

**根本原则**：如果两个模块互相依赖，它们可能该合并，或该提取共同部分。

## 9. `__pycache__` 与 .pyc 文件

```python
# 当你 import 一个模块时，Python 编译它为字节码
# 并缓存在 __pycache__/ 目录中
#
# module.py
#   → 编译
#   → __pycache__/module.cpython-312.pyc  ← 字节码缓存
```

下次导入更快。如果 .py 源文件的修改时间更新，Python 会重新编译。

## 10. 常用模块导入模式

### 模式1：单文件工具模块

```python
# utils.py
def helper1():
    pass

def helper2():
    pass

# 使用
from utils import helper1, helper2
```

### 模式2：目录式包

```python
# utils/
# ├── __init__.py     ← from utils import FileReader
# ├── file_utils.py   ← FileReader 的实际定义
# └── string_utils.py

from utils import FileReader
processor = FileReader()
```

### 模式3：大型项目结构

```python
# my_project/
# ├── my_project/
# │   ├── __init__.py
# │   ├── core/
# │   │   ├── __init__.py
# │   │   ├── engine.py
# │   │   └── config.py
# │   ├── utils/
# │   │   ├── __init__.py
# │   │   ├── helpers.py
# │   │   └── validators.py
# │   └── api/
# │       ├── __init__.py
# │       └── routes.py
# ├── tests/
# │   ├── test_core.py
# │   └── test_utils.py
# ├── setup.py / pyproject.toml
# └── README.md
```

## 12. 小结

1. **模块**：.py 文件，用 `import` 导入
2. **包**：含 `__init__.py` 的目录
3. **`if __name__ == "__main__"`**：区分直接运行和 import
4. **绝对导入优先**：清晰明了
5. **相对导入**：仅在包内部使用
6. **`__all__`**：控制 `from pkg import *` 行为
7. **避免循环导入**：延迟导入或重构
8. **`sys.path`**：找不到模块时检查搜索路径
